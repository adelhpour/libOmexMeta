Serializing RDF Graphs¶

Serializing is just a fancy term for writing the information in an RDF graph to a string or file. There are several syntaxes that are supported by libOmexMeta, including:

  • ntriples, turtle, rdfxml-xmp, rdfxml-abbrev, rdfxml, dot, json-triples, json, nquads, html

Writing to string¶

10    bqbiol:isPropertyOf local:EntityProperty0000 ;
11    bqbiol:isVersionOf <https://identifiers.org/OPB/OPB_00340> .
12
13local:EntityProperty0000
14    bqbiol:is <https://identifiers.org/uniprot/P84022> ;
15    bqbiol:isPartOf <https://identifiers.org/fma/FMA:24178>, <https://identifiers.org/fma/FMA:63877>, <https://identifiers.org/fma/FMA:70737>, <https://identifiers.org/fma/FMA:7163> .
16
17"""
18
19# remember that the default parser is "guess".
20# the parser will try to figure out which syntax is being used.
21# but if it doesn't guess well, you can use the format argument for `from_string`
22rdf = RDF.from_string(turtle_string)
23
24formats = [
