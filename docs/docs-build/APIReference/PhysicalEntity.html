OMEXMETA_DEPRECATED PhysicalEntity & addLocation (const std::string &where)

add a location to the PhysicalEntity.

The input string gets converted to a

Resource automatically. An arbitrary number of locations are allowed. The location is added to the back of a vector containing the Resources. Left most elements of this vector represent larger physiological locations which get smaller as the index of this vector increases via use of the “isPartOf” predicate. For example, a cytosolic TGFb molecule in a dermal fibroblast could have a locations vector first containing a reference to skin, then dermis, then fibroblast, then cytosol. This ends up being cytosol<isPartOf>fibroblast<isPartOf>dermis<isPartOf>skin.
Return

a reference to this Physical entity. Allows chaining together builder commands.

Parameters

  • where: The resource representing a location.

See Resource documentation for more details of valid input strings.

Deprecated in favour of PhysicalEntity::isPartOf
