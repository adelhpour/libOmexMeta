Developers Corner
=================

points to make:
    - Aim to be simple. Simple code, simple usage. Simple everything.
    - Do not use complicated C++ if possible. No templates or multiple inheritance
    - Aim to be coherent with SOLID principles
    - Always test your code.
    - Use Codecov and coverage tools to show you where your software is weak.
    - Put in a UML diagram.
    - Use docstrings and doctests
    - using travis
    - docker
    - writing the docs.
